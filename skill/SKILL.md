# SOIL English Teaching Deck Workflow（英文科專用）

以李俊儀教授的 SOIL 教學心法為基礎、專為**英文科**打造的教學簡報工作流。
六顆引擎、一條清楚的路徑：先想清楚教什麼，再安排怎麼教，最後才處理怎麼呈現。

> ⚙️ **本版特性（與數學科版的差異）**
> - 🎯 **英文科專用**：把原本的幾何圖形版型，換成**時態對照表、例句結構卡、對話雙欄、單字表、寫作前後對照、閱讀文本標注**等英文教學版型。
> - 🔑 **零金鑰**：移除 `draw`（gpt-image / OpenAI）生圖依賴。所有視覺改用 **PptxGenJS 原生**（色塊、表格、形狀）＋ **emoji**，老師不必申請任何 API 金鑰。
> - 📦 **產出**：`.pptx`（用 PptxGenJS 生成），完整六引擎。

## 第一步：判斷工作模式

收到請求後，先用一個問題判斷工作模式（只問這一題）：

> 請問你目前的狀況是？
> 1. 我有英文教材（課文、講義、單字表、文章），想做成教學簡報
> 2. 我有現成簡報（.pptx），想改得更好
> 3. 我只有一個主題（如「現在完成式」），想從頭開始
> 4. 我想定義一套簡報風格給之後反覆使用

| 回答 | 工作模式 | 跑哪幾顆引擎 |
|-----|---------|------------|
| 1 | 全流程模式 | 引擎 1→2→3→4→5→6 |
| 2 | 診斷修改模式 | 引擎 4（+3 補頁面角色）→ 修改 .pptx |
| 3 | 全流程模式 | 先協助蒐集內容 → 引擎 1→2→3→4→5→6 |
| 4 | 風格定義模式 | 引擎 5 |

若使用者直接貼了一段課文說「做成簡報」，可跳過這題直接進全流程，但進引擎一前先補問細節。

## 引擎一：SOIL 概念定位師

> 目標：抓出教學主軸，回答「這堂英文課要學生帶走什麼」

### 輸入
英文教材、課文、單字表、文章、或主題描述。

### 產出「概念定位稿」六項
1. **總概念**：最核心的 1 個概念（一句話）。英文科例：「現在完成式 = 過去動作 × 現在結果」。
2. **三個子概念**：支撐總概念。例：① have+p.p. 結構 ② 與過去式的差異 ③ 常見時間副詞（since/for/just/yet）。
3. **三個常見誤解**（英文科最關鍵，要對準學生易錯點）。例：①「完成式 = 過去式」②「yesterday 可配完成式」③「for/since 不分」。
4. **帶走一句話**：學生只記一句的話那句是什麼。
5. **最小事實包**：支撐理解所需最少規則 / 例句。
6. **投影片 vs 講稿**：哪些放投影片、哪些口頭補充。

### 輸出格式
```
## 概念定位稿
### 總概念
[一句話]
### 三個子概念
1. … 2. … 3. …
### 三個常見誤解（→ 正確理解）
1. [誤解] → [正確] …
### 帶走一句話
[那句話]
### 最小事實包
- 規則 / 例句 …
### 投影片 vs 講稿分配
| 內容 | 建議位置 | 原因 |
```
完成後呈現給使用者確認，再進下一步。

## 引擎二：SOIL 脈絡定位師

> 目標：安排理解節奏，不是排章節

### 輸入
引擎一的概念定位稿 + 教學對象 + 時間長度。

### 三段理解節奏
| 節奏 | 功能 | 佔比 |
|-----|------|------|
| **引起動機** | 讓學生知道這跟自己有關（真實語境、迷你對話、提問） | 15-20% |
| **維持注意** | 幫學生把規則組成關係（結構拆解、對照、例句歸納） | 60-70% |
| **喚起行動** | 讓理解變成能說、能寫、能判斷（口說 / 造句 / 填空 / 改錯） | 15-20% |

### 每段要說明
教學目的、放入內容、學生理解任務、建議頁數、入口與出口、常見設計錯誤。

### 輸出格式
```
## 脈絡定位稿
### 第一段：引起動機（第 1-N 頁）
- 教學目的 / 放入內容 / 學生理解任務 / 建議頁數 / 入口 / 出口 / 常見錯誤
### 第二段：維持注意（第 N-M 頁）
### 第三段：喚起行動（第 M-末 頁）
```
完成後呈現確認。

## 引擎三：SOIL 頁面架構師（英文科版型）

> 目標：給每一頁一個任務，不是列大綱

### 輸入
引擎二的脈絡定位稿。

### 可用頁面角色
| 角色 | 用途 |
|-----|------|
| 封面頁 | 標題、副標、講師 |
| 問題引入頁 | 用問題 / 真實語境讓學生好奇 |
| 迷思澄清頁 | 呈現常見英文錯誤，再翻轉為正確用法 |
| 比較頁 | 兩個用法並排對照（如過去式 vs 完成式） |
| 流程頁 | 步驟、句型生成順序、時間軸 |
| 分類頁 | 多個項目歸類（如時間副詞分組） |
| 例句頁 | 用具體例句說明抽象規則 |
| 數據頁 | 大數字、考點頻率、關鍵統計 |
| 總結頁 | 收攏重點、帶走一句話 |
| 行動頁 | 請學生造句 / 口說 / 填空 / 改錯 |
| 過渡頁 | 章節銜接（極簡，一句話） |

### 每頁要指定
頁碼、標題、頁面角色、本頁唯一主重點（只能一個）、建議頁型、學生理解任務、與前/後頁銜接、是否為關鍵頁（★）、**英文視覺版型需求（見下方 visual 規則）**。

### ▶ 英文教學視覺版型規則（visual 欄位，零金鑰）

英文科**不用幾何圖、不用 AI 生圖**。視覺一律用 PptxGenJS 原生元件（表格、色塊、形狀、emoji）。在頁面角色表用 `visual` 欄位標記要用哪一種版型：

| visual 版型 | 用途 | PptxGenJS 作法 |
|------------|------|---------------|
| `tense_table` | 時態 / 用法對照（過去式 vs 完成式…） | `addTable`，表頭用強調色，左右欄對比 |
| `sentence_card` | 例句結構卡：主詞/動詞/受詞分色塊 | 多個 `addShape(rect)` 橫排 + `addText`，S 青、V 橘、O 綠 |
| `dialogue` | 對話雙欄（A/B 來回，口說/聽力） | 左右兩欄文字泡泡（圓角 `roundRect`） |
| `vocab_list` | 單字表：單字 / 詞性 / KK音標 / 例句 | `addTable` 多欄；音標放 `[ ]` 純文字 |
| `writing_diff` | 寫作前後對照：原句 vs 修改 | 雙欄；原句紅色刪節、修改綠色；下方寫原因 |
| `reading_callout` | 閱讀文本 + 重點標注 | 大段文字 + 用色塊 highlight 目標句型 / 生字 |
| `word_family` | 字根 / 字族 / 搭配詞放射圖 | 中央 `roundRect` + 四周 `addText` + `addShape(line)` 連線 |
| `icon_grid` | 3–6 個並列要點，用 emoji 當圖示 | 網格 + emoji（如 ✅⚠️💡🎯📖✍️🗣️👂） |

**visual 欄位格式（在頁面角色表中標記）：**
```
第 N 頁：[頁面標題]
- 頁面角色：比較頁
- 唯一主重點：過去式只講過去；完成式強調到現在的結果
- visual:
    type: tense_table
    columns: ["", "過去式 (Simple Past)", "現在完成式 (Present Perfect)"]
    rows:
      - ["焦點", "過去那件事", "到現在的結果"]
      - ["例句", "I lost my keys.", "I have lost my keys."]
      - ["時間副詞", "yesterday, ago", "since, for, just, yet"]
    emphasis_col: 3        # 用強調色標示「要學生記住」的那欄
```

```
- visual:
    type: sentence_card
    sentence: "She / has finished / her homework."
    parts: ["She|S", "has finished|V", "her homework|O"]   # |後標記詞性，用於分色
    note: "have/has + 過去分詞 = 完成式"
```

> 一頁可同時用文字 + 一個 visual 版型。所有版型都是純 PptxGenJS，零金鑰、零外部渲染。

### 自我檢查
- 超過一半頁面同一種角色？→ 過於單調，調整。
- 「唯一主重點」其實含兩個概念？→ 拆兩頁。
- 每頁都有明確理解任務？→ 沒有就刪。
- **文法 / 對照類頁面是否善用 `tense_table` 或 `sentence_card`？**
- **例句頁是否用結構卡分色，而不是整段純文字？**

完成後呈現確認。

## 引擎四：SOIL 認知編修師

> 目標：用六個認知詞把每頁修成真正的教學頁

| 認知詞 | 做什麼 | 怎麼判斷 |
|-------|-------|---------|
| **降雜訊** | 刪掉無助理解的元素 | 拿掉它學生理解會變差嗎？不會就刪 |
| **區塊化** | 同類資訊歸在一起 | 一眼看得出幾個群組嗎？ |
| **增資訊** | 補上理解的橋梁 | 兩個用法之間的關係標記夠清楚嗎？ |
| **結構化** | 讓主從關係可見 | 看得出主規則 vs 例外嗎？ |
| **順脈絡** | 排出合理閱讀順序 | 視線會照你想的順序走嗎？ |
| **步驟化** | 讓理解一步步跟 | 句型生成該分幾步出現？ |

**全流程模式**：引擎三完成後，逐頁用六認知詞規劃呈現方式，作為引擎六指引。

**診斷修改模式**（使用者上傳 .pptx）：
1. 用 `python -m markitdown` 提取文字
2. 用 pptx 技能的 `thumbnail.py` 看視覺佈局
3. 逐頁用六認知詞診斷
4. 輸出「認知編修報告」（每頁六詞具體建議 + 修改優先級 高/中/低）

英文科常見提醒：例句不要超過 2 行；對照表欄位不超過 3；一頁只示範一個句型。

## 引擎五：SOIL 風格建構師

> 目標：把風格固定成規則，讓整份簡報一致。風格不該太早處理，骨架（引擎 1-4）定了才進來。

### 有參考簡報 / 圖片
逆向工程：整體氣質、標題位置與層級、配色系統、空白配置、強調方式、頁型偏好。

### 沒有偏好 → 依教學對象自動配置
| 教學場景 | 建議配色 | 氣質 |
|---------|---------|------|
| 兒美 / 國小 | Coral Energy / Teal Trust | 活潑、對比強 |
| 國中 | Teal Trust / Ocean Gradient | 清晰、活潑 |
| 高中 | Forest & Moss / Ocean Gradient | 清晰、穩重 |
| 成人 / 教師研習 | Sage Calm / Warm Terracotta | 溫暖、專業 |

### 風格規則（通用）
- 標題位置一致（全靠左或全置中，不混用）
- 章節色一致；同類頁型結構一致（所有比較頁長一樣）
- 每頁都有視覺元素（色塊 / 表格 / 形狀 / emoji），不做純文字頁
- 風格服務理解，不追求炫技；標題下不加裝飾底線

### 輸出：風格設定摘要
- **視覺基礎**：配色（主色 / 輔色 / 強調色 HEX）、字型搭配、標題與內文字級、佈局偏好。
- **英文科分色慣例**（給 `sentence_card` 用）：主詞 S = 青 `#00C6FF`、動詞 V = 橘 `#E8643A`、受詞 O = 綠 `#81C784`、補語/副詞 = 琥珀 `#FFB74D`。整份簡報統一。

> 本版不含 `image_policy`（無 AI 生圖）。

## 引擎六：SOIL 簡報總導演

> 目標：整合前五顆引擎，生成 .pptx（純 PptxGenJS、零金鑰）

### 輸入
概念定位稿、脈絡定位稿、頁面角色表（含 visual）、認知編修指引、風格設定。

### 技術執行：讀取 pptx 技能的技術文件
```bash
# 讀 PptxGenJS 完整 API 指南（路徑依你的環境，例如：）
cat ~/.claude/skills/pptx/pptxgenjs.md 2>/dev/null \
  || cat ~/.claude-skills/pptx/pptxgenjs.md
```

遵循 pptx 技能規範，套用林長揚 #1 文字比例（55/34/21/13）：

**字級標準**
| 元素 | 字級 | 用法 |
|------|------|------|
| 封面大標題 | 72–84pt bold | 封面、關鍵行動頁 |
| 一般標題 | 44–56pt bold | 每頁主訊息 |
| 副標 | 28–34pt bold | 主標下補充 |
| 內文 | 18–21pt | 條列、例句 |
| 註解 | 14–16pt | 署名、備註 |
| 徽章 | 18–22pt bold | 頁面角色標籤 |
| 強調 | 24–28pt bold | 強調色句子 |

**字型（含 fallback，零安裝）**
| 用途 | 建議 | Fallback（免安裝）|
|------|------|------|
| 中文標題 / 內文 | 源石黑體 / 微軟正黑體 | `Microsoft JhengHei` / `PingFang TC`（系統內建）|
| 英文例句 | `Segoe UI` / `Arial` | 系統內建 |

> 若沒裝特定字體就用系統內建，PptxGenJS 會自動 fallback，不影響產出。

**其他規範**
- `LAYOUT_WIDE`（16:9，13.333"×7.5"）、行距 1.2、最小邊距 0.5"
- 三明治結構：深色封面 + 結語、淺色內容頁
- 每頁都要有視覺元素（visual 版型或色塊）
- 固定 x 格點：左 `0.6/0.7`、雙欄 `5.1/7.2`、三欄 `0.6/4.85/9.1`

**頁面角色 → 佈局**
| 角色 | 佈局 |
|-----|------|
| 封面頁 | 深色全版 + 大標題居中 |
| 問題引入頁 | 大字問句 + 留白 |
| 迷思澄清頁 | 雙欄（❌ 錯誤用法 vs ✅ 正確用法）|
| 比較頁 | `tense_table` 或雙欄對照 |
| 流程頁 | 橫向箭頭步驟 / 時間軸 |
| 分類頁 | 2×2 / 2×3 網格（emoji 圖示）|
| 例句頁 | `sentence_card` 分色結構 |
| 數據頁 | 大數字 60–72pt + 小標籤 |
| 總結頁 | 條列收攏 + 帶走一句話 |
| 行動頁 | 深色背景 + 造句/口說/填空指引 |
| 過渡頁 | 全版色塊 + 一句話 |

### ▶ 英文視覺版型的 PptxGenJS 實作要點（零金鑰）

> 所有版型都用 PptxGenJS 原生 API，不需要任何圖檔、不呼叫 draw、不裝額外套件。

- **tense_table**：`slide.addTable(rows, {colW, border, fill})`。表頭列填強調色、白字；`emphasis_col` 那欄整欄淡色底，提示「要記住」。
- **sentence_card**：每個 part 一個 `addShape(pptx.ShapeType.roundRect, {fill: 該詞性色})` + 疊 `addText`，橫向等距排列；下方 `addText` 放 note。詞性配色用引擎五的分色慣例。
- **dialogue**：左右各一排圓角泡泡（A 靠左、B 靠右），`roundRect` + `addText`；可加小 emoji 🗣️/👂。
- **vocab_list**：`addTable`，欄位「單字 | 詞性 | 音標 [KK] | 例句」；單字粗體、音標放方括號純文字。
- **writing_diff**：雙欄；左欄原句（紅色、可加刪除線 `strike:true`），右欄修改（綠色）；底部一行寫修改原因。
- **reading_callout**：一個大 `addText` 放文本，另用半透明 `addShape(rect, {fill:{color, transparency:70}})` 疊在要 highlight 的句型 / 生字上，或用不同顏色 runs。
- **word_family**：中央 `roundRect`（字根），四周 4–6 個 `addText`，用 `addShape(line)` 連線。
- **icon_grid**：網格每格一個 emoji（大字級）+ 標題 + 一行說明。

### 生成 .pptx
1. 依頁面角色表逐頁寫 PptxGenJS 腳本（含上述 visual 版型）。
2. 用 Node 執行腳本產生 `.pptx`。
3. 用 pptx 技能的 `thumbnail.py` 產縮圖，逐頁預覽檢查（對齊、不重疊、例句不超過 2 行）。
4. 確認無誤再交付。

### 完成前檢查清單
